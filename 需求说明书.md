风铃草（BellFlower）跨端虚拟组网工具 Demo 设计需求与技术方案
一、项目概述
项目名称：风铃草 BellFlower
产品定位：轻量化、纯国内网络适配、跨端P2P虚拟局域网工具（对标贝锐蒲公英、极简版Tailscale）
核心场景：解决手机（iOS/安卓）与Windows电脑跨网络、跨内网、跨Wi-Fi/蜂窝数据的虚拟内网互通，无需公网IP、无需端口映射，自动打洞组网，实现设备互访、本地服务远程访问。
Demo 目标：完成最小可用产品，实现「设备入网、虚拟IP分配、P2P打洞、跨端互通、链路状态监控」五大基础核心能力，可稳定用于个人手机连电脑调试、本地Web服务远程访问、文件互通测试。
产品寓意：风铃草——随风联动、轻量联结、无声互通，契合工具轻量、自动组网、跨设备静默连通的核心特性，区别于蒲公英飘散式组网，主打稳定、低延迟、轻量化点对点互联。
二、核心功能需求（Demo 最小功能集）
仅保留落地必须功能，不做冗余开发，快速验证组网核心逻辑
1. 设备入网与身份认证
- 支持密钥组网模式（极简无账号体系，对标EasyTier，降低Demo开发成本）
- 任意设备输入相同组网密钥，自动加入同一虚拟局域网
- 设备自定义备注名称（Windows电脑、iPhone、安卓手机）
- 设备在线/离线状态实时展示
2. 虚拟网络核心能力（核心重点）
- 客户端自动获取虚拟内网IP（统一网段，如 10.144.0.0/24）
- NAT穿透：支持多层家用路由、校园网、公司内网、4G/5G蜂窝网络打洞
- P2P直连优先，打洞失败自动降级中继（国内轻量化中继逻辑）
- 跨设备虚拟IP互通：Ping通、端口访问、HTTP本地服务远程调用
3. 跨端互联能力（手机 & 电脑双向互通）
- 手机（4G/5G/外网）访问电脑本地服务：虚拟IP:端口（Web服务、接口调试、本地后台）
- 电脑访问手机本地服务、调试移动端本地接口
- 基础连通性测试：内置Ping测试、链路延迟检测
4. 基础客户端能力
- 后台常驻虚拟网卡服务，开机/自启动可选
- 网络状态展示：在线状态、虚拟IP、延迟、连接模式（P2P/中继）
- 手动重连、断网自动重连、密钥重置
5. Demo 非功能需求
- 轻量化：客户端体积小、低内存占用，手机端无后台耗电冗余
- 国内适配：规避境外节点，纯国内网络链路，无高延迟、无访问超时
- 高兼容：支持Win10/Win11、iOS15+、安卓10+主流设备
- 高稳定：断网自动重连、切换网络（Wi-Fi/蜂窝）自动适配链路
三、非需求（Demo 阶段不做，控制开发量）
- 不做账号注册、云端用户系统、付费体系
- 不做权限管控、多团队、多子网复杂路由
- 不做文件传输、远程桌面、监控录像等附加功能
- 不做企业级审计、日志溯源、等保合规能力
四、整体架构设计（极简三层架构）
采用轻量化中心化控制 + P2P数据传输架构，兼顾开发难度和稳定性，适配Demo快速落地
1. 轻量控制服务端：仅负责设备发现、密钥校验、节点信息同步、打洞协商（不传输业务数据）
2. P2P数据传输层：设备之间点对点直连，业务数据不经过服务端，低延迟
3. 多端客户端：Windows客户端、移动端（iOS/安卓）客户端，创建虚拟网卡、实现网络互通
核心优势：服务端压力极小，可本地部署/轻量云服务器部署，完全适配个人Demo项目。
五、全套技术栈（适配新手开发、轻量化、开源免费、国内可用）
1. 底层网络核心（组网核心）
- 核心协议：WireGuard（轻量、高速、安全，Tailscale/蒲公英同款底层，开源无版权）
- NAT打洞库：STUN 协议（国内公共STUN服务器，免费可用，无需自建）
- 虚拟网卡驱动：TUN/TAP 跨平台虚拟网卡
选型理由：WireGuard 代码极简、功耗低、穿透成功率高，非常适合手机+电脑轻量化组网，开发成本远低于自研组网协议。
2. 服务端技术（轻量自研控制服务）
- 开发语言：Golang（高性能、跨平台、单文件部署、并发强，适配网络服务）
- 通信协议：WebSocket（实时设备状态同步、打洞协商）+ HTTP（基础接口）
- 数据存储：SQLite（本地文件数据库，无需部署MySQL，零运维）
- 部署方式：单二进制文件，Windows/Linux一键启动
3. Windows 客户端技术栈
- 界面框架：Qt / WPF（轻量化桌面GUI，快速开发可视化界面）
- 网络内核：内嵌WireGuard-Go 客户端内核
- 能力支持：虚拟网卡挂载、状态监控、密钥配置、自动重连
4. 移动端技术栈（iOS + 安卓统一方案）
- 跨端框架：Flutter（一套代码编译双端，大幅降低开发量）
- 组网内核：集成开源 WireGuard 移动端 SDK
- 核心能力：密钥入网、虚拟IP展示、链路检测、后台保活
5. 辅助工具 & 环境
- 版本控制：Git + GitHub/Gitee
- 构建工具：Go Build、Flutter Build
- 测试工具：Ping、本地Nginx服务、端口监听工具
六、Demo 运行流程（极简业务逻辑）
1. 部署风铃草轻量服务端，启动节点发现服务；
2. 电脑、手机客户端输入同一组网密钥，连接服务端完成入网；
3. 服务端同步所有在线设备节点信息，发起NAT打洞协商；
4. 设备之间建立P2P直连，各自生成唯一虚拟内网IP；
5. 手机/电脑通过虚拟IP互相访问本地端口、服务，完成跨端组网互通。
七、项目优势（对标蒲公英/同类工具）
- 极简无广告：Demo纯净无广告、无弹窗、无付费诱导
- 轻量化低功耗：手机端后台占用极低，不耗电、不占内存
- 纯自主可控：无第三方厂商劫持、无设备数量硬性限制（Demo阶段）
- 国内网络优化：摒弃境外节点，适配国内运营商网络，穿透成功率、延迟优于Tailscale
八、迭代规划（Demo→正式版）

- V1.1：增加设备在线日志、链路质量检测、自动优选线路
- V1.2：增加简易Web管理面板、自定义网段、中继节点优化
- V1.3：新增基础文件互传、内网设备扫描能力
